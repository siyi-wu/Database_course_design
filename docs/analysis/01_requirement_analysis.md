# 实验室设备管理系统需求分析

## 1. 系统目标

实验室设备管理系统面向学校实验室设备借用与维护场景，目标是统一管理设备类别、实验室房间、用户、设备台账和借还记录，提高设备使用透明度，减少人工登记错误，并为设备统计、状态追踪和后续维护提供数据支撑。

## 2. 用户角色

- 学生：查询可用设备，提交借用和归还。
- 教师：查询设备状态，借用教学或科研设备。
- 实验室管理员：维护设备台账、处理借还、调拨设备、查看统计结果。
- 系统维护人员：维护数据库对象、备份数据、优化性能。

## 3. 功能需求

- 设备类别管理：新增、修改、查询设备类别。
- 实验室房间管理：维护房间名称、位置和管理员。
- 用户管理：维护借用人姓名、角色和联系方式。
- 设备台账管理：维护设备名称、类别、房间、状态、价格和购置日期。
- 借用管理：登记设备借出，自动检查设备是否可借。
- 归还管理：登记实际归还日期，自动恢复设备状态。
- 查询统计：按类别、状态、用户、房间查询设备和借用记录。
- 数据一致性：通过外键、触发器和存储过程保证设备状态与借还记录一致。

## 4. 非功能需求

- 数据完整性：主键、外键和业务触发器共同保证数据正确。
- 性能要求：对高频查询建立二级索引，降低全表扫描。
- 可维护性：数据库对象命名清晰，SQL 脚本按职责拆分。
- 可扩展性：后续可增加维修记录、预约记录、权限管理和报表模块。

## 5. 业务规则

- 每台设备必须属于一个设备类别。
- 每台设备必须放置在一个实验室房间。
- 每条借用记录必须对应一个有效用户和一台有效设备。
- 维修中或已报废设备不能借出。
- 同一设备同一时间只能存在一条未归还借用记录。
- 借用成功后设备状态自动变为 `Borrowed`。
- 归还成功后如果不存在其他未归还记录，设备状态自动恢复为 `Available`。
