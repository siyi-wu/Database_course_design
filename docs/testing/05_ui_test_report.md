# UI 功能测试报告

## 1. 测试环境

- 前端：React + Vite
- 后端：Express + mysql2
- 数据库：MySQL 8.0 / InnoDB
- 数据库名：`<database_name>`
- 前端地址：`http://localhost:5173/`
- 后端地址：`http://localhost:3001/`

## 2. 测试范围

本测试覆盖实验室设备管理系统 UI 中的总览、设备、借还、用户、房间、类别和记录模块，重点验证 UI 能否正确调用后端接口，并通过后端访问真实数据库。

## 3. 测试用例

| 编号 | 测试项 | 操作 | 预期结果 | 状态 |
| --- | --- | --- | --- | --- |
| UI-01 | 后端连接 | 访问总览页 | 显示数据库统计数据和连接状态 | 通过 |
| UI-02 | 设备查询 | 按状态、类别、房间筛选设备 | 表格显示符合条件的设备 | 通过 |
| UI-03 | 新增设备 | 输入设备名称、类别、房间、状态、价格、购置日期并保存 | 设备写入 `equipments` 表 | 通过 |
| UI-04 | 编辑设备 | 修改设备价格或状态 | 数据库中对应设备更新 | 通过 |
| UI-05 | 删除设备 | 删除未被借用记录引用的演示设备 | 设备被删除；被引用设备由外键阻止删除 | 通过 |
| UI-06 | 用户维护 | 新增、查询、编辑用户 | `users` 表正确变化 | 通过 |
| UI-07 | 房间维护 | 新增、查询、编辑实验室房间 | `labrooms` 表正确变化 | 通过 |
| UI-08 | 类别维护 | 新增、查询、编辑设备类别 | `categories` 表正确变化 | 通过 |
| UI-09 | 借出设备 | 选择可用设备和用户，点击借出 | 调用 `sp_borrow_equipment`，新增借用记录，设备状态变为 `Borrowed` | 通过 |
| UI-10 | 重复借出 | 对同一未归还设备再次借出 | 触发器拒绝操作，UI 显示错误提示 | 通过 |
| UI-11 | 归还设备 | 选择未归还记录，点击归还 | 调用 `sp_return_equipment`，写入归还日期，设备状态恢复为 `Available` | 通过 |
| UI-12 | 记录查询 | 搜索记录编号、设备名或用户名 | 显示对应借用流水和状态 | 通过 |

## 4. 错误反馈验证

UI 对以下错误场景能够显示数据库返回的错误信息：

- 设备编号或用户编号为空。
- 设备不存在。
- 用户不存在。
- 维修或报废设备不能借出。
- 同一设备存在未归还记录，不能重复借出。
- 借用记录不存在。
- 借用记录已经归还，不能重复归还。

## 5. 测试结论

UI 能够完成课程设计要求中的基本增删改查、简单查询、复杂查询展示、借用归还业务操作和错误反馈。界面操作与数据库触发器、存储过程、函数保持联动，能够作为验收演示入口。
